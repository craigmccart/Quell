## 2024-05-24 - XSS via Webview HTML interpolation
**Vulnerability:** The VS Code webview Content Security Policy allows 'unsafe-inline' scripts. Filenames shown in the UI (`f.file`) and error messages were being directly interpolated into HTML strings and `onclick`/`onkeydown` handlers in `SidebarProvider.ts` without proper HTML or JavaScript encoding.
**Learning:** Even though the filenames are sourced from the local filesystem, an attacker who can trick a user into cloning a repository with a maliciously crafted filename (e.g. `"><script>...</script>`) could execute arbitrary JavaScript within the webview context. Since event handlers require double escaping (JavaScript string escaping inside an HTML attribute), simply escaping quotes is insufficient.
**Prevention:** Always rigorously HTML-escape all dynamic data before interpolation into Webview HTML strings. For inline JavaScript event handlers (like `onclick`), the data must be properly JS-escaped (e.g., using `JSON.stringify`) AND then HTML-escaped.
