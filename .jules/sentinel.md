## 2025-02-15 - Prevent Arbitrary VS Code Command Execution from Webview via Missing Validation
**Vulnerability:** The VS Code webview `onDidReceiveMessage` event handler in `src/SidebarProvider.ts` blindly passed `data.command` directly to `vscode.commands.executeCommand(data.command)` without validation, allowing a malicious payload from the webview to execute arbitrary VS Code commands.
**Learning:** Webviews execute in a sandbox but can communicate with the extension via `postMessage`. When taking untrusted strings from a webview message and passing them to powerful APIs like `vscode.commands.executeCommand`, explicit validation (e.g., prefix checking, allowlisting) is necessary to prevent injection. The webview has 'unsafe-inline' enabled for scripts and styles, which makes an XSS vulnerability more critical. If an attacker injects a script into the webview, they can use `vscode.postMessage` to run dangerous commands.
**Prevention:** Strictly validate or whitelist incoming command strings from webviews before invoking execution contexts. Since legitimate Quell commands are prefixed with `quell.`, enforcing that `command.startsWith('quell.')` prevents executing potentially harmful built-in or third-party commands.
