## 2025-04-03 - [CRITICAL] Arbitrary Command Execution via Webview Messages
**Vulnerability:** The webview message handler in `src/SidebarProvider.ts` passed untrusted `data.command` directly to `vscode.commands.executeCommand` without any validation. An attacker who could execute malicious JavaScript within the webview context (e.g. via XSS) could execute arbitrary VS Code commands, potentially leading to arbitrary code execution or unauthorized access to the host machine.
**Learning:** Never trust input originating from webviews, even if `enableScripts` is only used for internal UI interaction. All IPC messages from a webview should be considered untrusted input. The assumption that only expected UI elements will trigger commands is false.
**Prevention:** Strictly validate or whitelist all commands received from webviews before passing them to internal execution APIs. Always ensure the `command` is a string and that it strictly matches a predefined list or starts with an expected, safe extension prefix (e.g. `quell.`).
