## 2024-05-24 - Webview HTML Injection / XSS
**Vulnerability:** Unsanitized workspace file paths (e.g. `f.file`) were being directly interpolated into HTML attributes (`title="..."`, `onclick="..."`) and element content (`<span...>${f.file}</span>`) in the VS Code Webview HTML generation string.
**Learning:** Even if CSP `script-src` includes `unsafe-inline`, any string values that can be influenced by the environment (like workspace paths which can be maliciously crafted) and injected into the HTML string must be carefully escaped. Because the Webview evaluates HTML strings directly, an unescaped value inside double quotes will break out of the HTML attribute and execute arbitrary javascript (e.g. `"<script>...</script>.txt"`).
**Prevention:** Whenever manually constructing an HTML string for a Webview, explicitly run all dynamic text parameters through an `escapeHtml` function replacing `<, >, &, ', "` before interpolating them into the HTML string. If interpolating inside javascript event handlers (e.g. `onclick`), ensure string escaping for javascript is done *before* escaping for HTML.
