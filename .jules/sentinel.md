## 2024-05-24 - Fix Cross-Site Scripting (XSS) in Sidebar Webview
**Vulnerability:** The Sidebar Webview (`src/SidebarProvider.ts`) allowed Cross-Site Scripting (XSS) because workspace filenames were injected directly into the HTML without sanitization, and the Content Security Policy allowed `unsafe-inline` scripts. A malicious repository with crafted filenames (e.g. containing `<script>`) could execute arbitrary code in the webview context.
**Learning:** Even internal extensions where "user input" is implicitly trusted (like workspace filenames) must be treated as untrusted data. When constructing HTML strings manually, data must be strictly escaped to prevent XSS, especially if the CSP is permissive regarding inline scripts.
**Prevention:** Always use a dedicated HTML escaping function (like `escapeHtml`) when interpolating dynamic data into HTML strings. Ensure that data passed into inline JavaScript handlers (e.g., `onclick`, `onkeydown`) is appropriately escaped for both JavaScript strings and HTML attributes.
