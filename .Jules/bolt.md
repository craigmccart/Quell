## 2024-04-14 - Optimize SecretScanner replacement and lookup paths
**Learning:** In high-frequency text replacement functions (like `SecretScanner.redact`), replacing O(N) linear iteration for lookups with O(1) reverse-lookup Maps (e.g., `secretToPlaceholder`) significantly improves execution speed. Additionally, replacing `.split(delimiter).join(replacement)` with `String.prototype.replaceAll()` is faster and more memory-efficient. When the replacement string in `replaceAll` is arbitrary, it must be passed as a callback `() => replacementValue` to prevent special replacement patterns from executing.
**Action:** Always use reverse-lookup Maps for O(1) lookups in performance-critical paths instead of iterating over Map entries. Use `replaceAll` with a callback instead of `.split().join()` for high-frequency string replacements.
